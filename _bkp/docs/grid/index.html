Grid

How to use our grid classes and breakpoints.

Settings

All of the settings used to generate the grid.

Location: /src/components/grid/_grid-settings.scss

Feel free to tweak all of these values but don't change any of the generator functions. There is a warning on the file where those are suposed to be left alone.

The name of the variables are pretty self explanatory.

⚠️ The breakpoint variables are generated from the range variables.

$column-gutter: 16px;
$row-width: 1144px;
$total-columns: 12;

$xsmall-range: (0, 24.9375em); // 399px
$small-range: (25em, 39.9375em); // 639px
$medium-range: (40em, 63.9375em); // 1023px
$large-range: (64em, 71.4375em); // 1143px
$xlarge-range: (71.5em, 89.9375em); // 1439px
$xxlarge-range: (120em, 99999999em);

The Grid

With the settings in place and grid generated, it's time to use the classes.

Location: /src/components/grid/grid.scss

Wrappers

Wrapper are used to create a row that don't have any columns

Usage

.wrapper
  h1 Reprehenderit
  p Aute consectetur aute consectetur non ipsum magna esse ea. Labore

Rows

⚠️ You HAVE to use a row anytime you need columns.

Nested rows required a modifier class .row--deep.

To remove extra padding from columns and allow for them to sit flush to one another - we can add the modifier class .row--gutterless to the parent row.

Usage

.row.row--gutterless
  .column
    p Aute consectetur aute consectetur non ipsum magna esse ea.
  .column
    p Aute consectetur aute consectetur non ipsum magna esse ea.
  .column
    p Aute consectetur aute consectetur non ipsum magna esse ea.
    .row.row.row--deep
      .column
        p Nested row and columns
      .column.column--6
        p Nested row and columns

Columns

Columns are flexible by default. They will expand to fill the space available.

Usage

.row
  .column
    p Aute consectetur aute consectetur non ipsum magna esse ea.
  .column
    p Aute consectetur aute consectetur non ipsum magna esse ea.

Breakpoints

These can be used anywhere on the website, just gotta remember to import the grid settings.

Notice how we start from mobile and we use the breakpoints as the viewport grows, not the other way around.

TLDR' MOBILE FIRST.

Usage

@import './include/grid/grid-settings';

.my-div {
  background-color: red;

  @media #{$medium-up} {
    background-color: yellow;
  }

  @media #{$large-up} {
    background-color: green;
  }

  @media #{$xlarge-up} {
    background-color: black;
  }
}
